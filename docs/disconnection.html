Why Three-Way Handshake?

Understanding the design rationale

Problem: Single Request Disconnection

The original implementation used a simple "quit" message, which had several issues:

  • No confirmation that server received the disconnection request
  • Potential data loss if messages were in flight
  • Resource cleanup timing was unpredictable
  • Race conditions between client and server cleanup

Solution: Three-Way Handshake

Inspired by TCP's connection termination, our protocol ensures:

  • Mutual Agreement: Both sides acknowledge disconnection intent
  • Ordered Shutdown: Clear sequence prevents race conditions
  • Reliable Cleanup: Resources freed only after confirmation
  • Timeout Recovery: Automatic fallback for network issues
